Provides support for Conditions to the WindowTester runtime.

UI Tests are made robust using Conditions. Conditions implement the {@link com.windowtester.runtime.condition.ICondition} interface and are commonly used to cause tests to pause until some condition becomes true (for instance once a file is created on the filesystem). Conditions can also be registered with the runtime's {@link com.windowtester.runtime.condition.IConditionMonitor} and used to trigger {@link com.windowtester.runtime.condition.IHandler}s. Special Eclipse-specific conditions can be found in the com.windowtester.runtime.swt.condition.eclipse.package. The com.windowtester.runtime.swt.condition.shell package provides conditions and handlers tailored for monitoring SWT Shell conditions.
